網站建置過程

本網站整體架構是基於 Quarto 框架建構而成。Quarto 可以透過 Markdown 語法，快速生成網站，避免了 JavaScript、HTML、CSS 等前端語法的額外學習成本，我希望將重點放在文章內容的呈現，而非網站技術細節。因此先求有再求好，未來若 Quarto 框架因為網站功能需求而無法滿足，我會再透過其他前端技術進行網站改寫。

在開發的過程中，由於我是商管背景的學生，僅熟悉 Python、SQL 等程式語言，對於前端技術略有涉略但不夠熟悉，因此整個網站是透過 Vibe Coding 的方式，由我和 Claude、Gemini 共同完成，風格則參考了 OpenAI 和 Google 的設計邏輯。在過程中，我體會到 AI Agent 在程式開發上的驚人進展，若沒有 AI Agent 的協助，我可能需要花費數倍的時間才能完成網站的建置。

同時我也深刻意識到，在 AI 的時代 Know-Why 或許比 Know-How 更加重要。下一章節我想進一步分享我對於網站設計的核心考量，以及為何我想把網站設計成現在所看到的樣子。

設計核心考量

版面配置與閱讀體驗

在螢幕版面設計上，最基本的原則是：「網站內容必須絕對置中」，以維持視覺的和諧。

我時常閱讀技術部落格或是官方文件，其中最讓我受不了的就是網站內容因為側邊欄 (Sidebar) 或是目錄 (TOC) 的存在，被迫偏向螢幕的左側或右側。

因此本網站的設計排除了常見但會改變頁面重心的干擾。只有在螢幕寬度足夠大的情況下，才會在兩側展開 Sidebar 與 TOC；當螢幕寬度不足以完美置中主內容時，這兩個元件會被隱藏，減少視覺干擾，並確保文章內容始終維持在瀏覽器的正中央。

Copy Page 功能

文章標題右下方的 Copy Page 按鈕可一鍵將全文複製至剪貼簿。這個功能是參考自許多官方技術文件的設計，方便讀者將網站內容丟給 AI 進行摘要總結或是依內容回答相關問題。這對我來說非常實用，因此我也將這個功能加到我的網站中。

低彩度與高對比的色彩

在顏色的選擇上，為了降低長時間閱讀文字、數據所帶來的視覺疲勞感，整體避免使用純黑色。標題與內文選用不同深淺的深灰色來建立文字層次。背景部分維持純白，並利用微灰色來特別標示程式碼或特定視窗區塊。

字體排版設計

技術文章中經常包含大量的中英文混排與程式碼。因此網站建立了一個四類的字體系統：

  1. 英文與數字：
    選用 DM Sans，此字體具備現代感且閱讀上較為平穩。
  2. 繁體中文：
    採用 Noto Sans TC (思源黑體)，具備閱讀舒適性，且在風格上能與 DM Sans 搭配。
  3. 程式碼：
    使用 JetBrains Mono，具備字體連字功能 (Font Ligatures)，例如能讓 <= 變成 <=，提升程式碼可讀性。
  4. 數據與表格：
    使用 Roboto Mono。每個數字字元寬度完全相同，能確保表格內的數據精準對齊，避免視覺錯位。

字體大小與行高設定

內文基準字體大小設定為 16px，符合標準設定。另外行高方面，相較於常規的 1.5 會造成中文內容閱讀上過去擁擠，我將網站行高設定為 1.8，減少長文閱讀時的視覺壓迫感。

最大閱讀寬度

為了維持最佳的閱讀節奏，我將文章內文的最大寬度限制在 672px。相較於讓內容在寬螢幕下無限制延伸，適度的橫向空間限制能縮短視線來回掃描的距離，有效降低長文閱讀時的疲勞感。

實作成果

為了後續寫作與維護的效率，在此記錄本站常用的 Quarto 語法與自定義組件範例作為 Syntax Cheat Sheet。讓我之後在撰寫新文章時，可以直接參考或複製此處的語法。

訊息提示區塊

原生的 Quarto Callout 帶有比較沈重的底色，因此我透過 SCSS 將其改成類似 GitHub Alerts 的輕量化風格：只有左邊一條裝飾線。寫作時，只需修改大括號內的標籤 (note, tip, warning, important) 就能切換。

頁籤切換

當我需要呈現多組平行的資料（例如兩組對照表），卻又不想讓網頁拉得太長時，我會使用 Tabsets，設計風格參考 OpenAI 的 Pill Tabs。

語法範例

::: {.panel-tabset}
## 方案 A
這是方案 A 的詳細說明與表格配置。在頁籤下方可以自由放入文字、資料或程式碼。

## 方案 B
這是方案 B 的獨立內容，點擊上方的「方案 B」就可以無縫切換，不會重新載入網頁。

FAQ 折疊選單

我有時會希望在文章的中間，穿插一些較為輕巧的問答，但又不希望像 Callout 那樣具有強烈的視覺強調感，為此我手刻了這組 <details> 與 <summary> 的折疊外觀。

語法範例

<details class="faq-toggle">
<summary>這個網站是用什麼技術架設的？</summary>
<div class="faq-answer">

主要使用 Quarto 作為基礎的靜態網站生成器，並客製化了大量的 SCSS 與 HTML 模板來達到目前的設計效果。

</div>
</details>

三線表

學術界經常使用的「三線表」，我也把它作為網站表格的設計風格。在撰寫 Markdown 表格時不需要額外的 class，它會自動去掉多餘的網格線，只保留最俐落的三條水平線。同時為了方便進行同列比較，當滑鼠 hover 到不同資料列時，會呈現淡灰色的背景。

總結

這篇文章記錄了網站建置的過程與核心設計考量，並將常用的 Quarto 語法與自訂元件整理成速查表 (Cheat Sheet)。這份紀錄不僅能確保未來產出的文章在版面與視覺風格上保持一致，也能大幅節省每次尋找排版語法的時間，讓我能將更多精力集中在內容本身的撰寫上。
